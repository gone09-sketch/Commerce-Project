# 실시간 커머스 플랫폼  
---

## 1. 프로젝트 소개 
**프로젝트명: 실시간 커머스 플랫폼**
<br>
<br>✔️ 커머스 플랫폼의 특성을 고려한 클래스 설계 
<br>✔️ 각 객체의 역할과 책임을 고민하여 설계
<br>✔️ 필수과제 메서드 분리 완료 
<br>✔️ 도전과제 장바구니 구현 완료 / 메서드 분리 X 

<br>

## 2. 프로젝트 구조 
```plaintext

Commerce_Project/
├── src/
│   ├── Commerce/          
│       ├── Main.java              # start 호출, 객체생성  
│       ├── CommerceSystem.java    # 흐름 컨트롤러(start 실행), 입출력 
│       ├── Category.java          # 카테고리 관리(카테고리명), 상품리스트 관리
│       ├── Product.java           # 상품 관리(상품명, 가격, 설명, 재고수량)
│       ├── Customer.java          # 고객 관리(고객명, 이메일, 등급)
│       ├── Cart.java              # 장바구니 전체 관리(장바구니 아이템 리스트, 장바구니 내역, 장바구니 삭제)
│       ├── CartItem.java          # 장바구니에 추가된 아이템(상품+추가수량) 관리


```
---

## 3. 주요 기능 
✔️ 카테고리 메뉴에서 사용자 입력 받기 
<br>✔️ 상품 메뉴에서 사용자 입력 받기 
<br>✔️ 선택한 상품 장바구니에 추가 및 취소 
<br>✔️ 장바구니 내역 확인 및 주문 취소 
<br>✔️ 주문 확정 후 재고 변경 확인 


---
## 4. 실행 방법
1️. [ 실시간 커머스 플랫폼 메인 ] 메인 메뉴의 카테고리 선택 // 0 입력 시, 프로그램 종료 
<br>
<br>2. 선택한 카테고리 안의 상품리스트 확인 후, 상품 선택 // 0 입력 시, 메인메뉴로 이동 
<br>
<br>3. 선택한 상품을 장바구니에 추가 선택 시, 장바구니에 추가되며 메인메뉴로 이동되며 하단에 [ 주문 관리 ] 메뉴가 함께 출력
<br>(3-1). 취소 선택 시, 장바구니 추가가 취소되며 메인메뉴로 이동 
<br>
<br>4. [ 주문 관리 ] 메뉴에서 4. 장바구니 확인 선택 시, [ 장바구니 내역 ] & [ 총 주문 금액 ] & 주문 확정 선택 가능 
<br>(4-1). [ 주문 관리 ] 메뉴에서 5. 주문 취소 선택 시, 장바구니 내역에 있던 모든 상품들이 삭제되며 메인메뉴로 이동
<br>
<br>5. 주문 확정 선택 시, 주문이 완료되며 재고 업데이트 확인이 가능하며 다시 메인메뉴로 이동 

<br>
